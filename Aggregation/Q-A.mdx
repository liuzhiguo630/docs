  <Accordion title="Q:无限卡和积分有什么区别" icon="money-bill-wave" iconType="regular">
    无限卡和积分是本平台的两种账户资源，分别对应了官网的包月计费和 API 的按量计费 镜像站 1:1 还原了官方网站，同样包括了官网**按月付费**的无限卡消耗

    聚合站本质上是调用的各种模型的 API 来我们的前端网站使用，因此同样采用了**按量计费**的积分制消耗

    不过用户大可放心，无论是哪种计费方式，都远比官方价格更加便宜
  </Accordion>
  <Accordion title="Q:为什么对话越长，消耗积分越多" icon="face-confused">
    AI 是没有记忆的，因此与 AI 进行多轮对话的原理本质上是：<u>每一次对话时都会将此前的对话记录全部发给 AI</u>，随着对话轮数越来越多，发给 AI 的请求 Token 自然也会增多，所以会导致消耗逐渐增加

    但用户可以通过选择模型的下拉菜单中的**附带历史消息条数**来控制每一次对话时都只发送最近多少条消息，来合理控制模型的记忆和消耗
  </Accordion>
  <Accordion title="Q:镜像站常见报错" icon="brake-warning">
    用户可能经常在 Claude 镜像站看见 **「you have sent too many message....」** 或类似的报错，这其实是因为由于采用了订阅制度，所以任何 AI 的官方网站都是有着**速率限制**的：

    即一个账户在单位时间内能发送的消息有限不过本平台支持**新开对话**之后就能切换到另一个可用的账户上去，所以如果当您当前对话受限了，新开一个对话即可解决问题

    <Note>
      请注意！如果看见了要求您升级如 Plus，或 Max 订阅等提示，请不要点击，因为那是在提醒您当前对话背后的账户升级订阅，而您在本平台的唯一使用凭证是登录码，与官网提示并无联系
    </Note>
  </Accordion>
  <Accordion title="Q:为什么聚合站无法上传文件" icon="upload">
    当前的 AI 模型只具备文本和图像的处理能力，所以上传文件并不是模型原生的能力

    负责上传和解析文件的是网站，而模型本质上仍然是在处理文件里的文本和图像

    您可以将我们的 API 接入支持解析文件的第三方平台「如 Chatbox」前去使用，或是手动将文件中的内容传给 AI，将会收获完全一样的效果，因为 AI 本质就是这样在工作

    <Tip>
      我们将很快会支持上传文件功能，请关注近期更新页面～
    </Tip>
  </Accordion>
  <Accordion title="Q:为什么我邀请人没有返回积分" icon="star-of-david">
    通过您的邀请链接**首次加入我们**的新用户将会与您建立绑定关系

    您可以通过点击**推荐列表**里的内容查看与用户的绑定关系，如果该用户此前已经访问过我们网站了，由于他初次访问我们平台时就会自动分配登录码，因此该登录码是不会与您建立绑定关系的
  </Accordion>
